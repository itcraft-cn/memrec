# 安全要点总结

## 核心安全特性

### 1. 🛡️ 模型文件完整性保护
- **SHA256哈希验证**: 所有下载的模型文件都经过哈希校验
- **防篡改**: 防止恶意镜像服务器提供篡改的ONNX模型
- **可选绕过**: `--skip-hash-verify` 需要用户明确知晓风险

### 2. 🔒 Git供应链安全
- **仓库白名单**: 仅允许官方和可信镜像仓库
- **防止注入**: 阻止恶意Git仓库的代码注入攻击
- **可选绕过**: `--allow-any-repo` 需要用户明确知晓风险

### 3. 🔐 最小权限原则
- **用户模式服务**: 不以root或系统权限运行
- **文件权限**: `~/.memrec/config.toml` (600), 数据目录 (700)
- **网络隔离**: 仅本地Unix socket通信，无网络暴露

### 4. ⚠️ 安全透明性
- **明确警告**: 危险选项有清晰的安全警告
- **文档完整**: 安全最佳实践和风险说明
- **审计日志**: 关键操作记录便于审查

## 命令行安全选项

### 安全默认配置
```bash
# 最安全：使用默认配置
mr-install
```

### 可控风险选项
```bash
# 使用可信镜像（hf-mirror.com是官方镜像）
mr-install --use-hf-mirror

# 自定义镜像（需确保来源可信）
mr-install --mirror-base-url "https://your-trusted-mirror.com"
```

### 高风险选项（需明确知晓风险）
```bash
# 跳过哈希验证（可能下载恶意模型）
mr-install --skip-hash-verify

# 使用任意Git仓库（可能执行恶意代码）
mr-install --allow-any-repo --repo-url "https://example.com/repo"
```

## 安全最佳实践

### 安装时
1. **验证来源**: 从官方渠道下载安装器
2. **检查哈希**: 验证安装器文件的完整性
3. **使用默认**: 除非必要，使用默认安全配置

### 运行时
1. **权限检查**: 确保 `~/.memrec/` 目录权限正确
2. **日志监控**: 定期检查服务日志
3. **定期更新**: 及时应用安全更新

### 更新时
1. **备份数据**: 更新前备份记忆数据
2. **测试验证**: 更新后验证功能正常
3. **审查变更**: 查看更新日志中的安全修复

## 攻击防护能力

| 攻击类型 | 防护机制 | 有效性 |
|----------|----------|--------|
| 恶意模型注入 | SHA256哈希验证 | ✅ 高 |
| 供应链攻击 | Git仓库白名单 | ✅ 高 |
| 权限提升 | 用户模式服务 | ✅ 中 |
| 命令注入 | 硬编码服务配置 | ✅ 高 |
| 数据篡改 | 文件权限控制 | ✅ 中 |

## 应急响应

### 怀疑被攻击时
1. **立即停止**: `systemctl --user stop memrecd`
2. **检查日志**: `journalctl --user -u memrecd`
3. **验证文件**: 重新下载并验证模型文件
4. **联系支持**: security@itcraft.cn

### 发现漏洞时
1. **报告**: GitHub Security Advisory
2. **隔离**: 停止受影响服务
3. **更新**: 应用安全补丁
4. **通知**: 告知其他用户

## 安全设计原则

1. **默认安全**: 安全选项默认启用
2. **最小权限**: 仅授予必要权限
3. **防御深度**: 多层安全防护
4. **透明可信**: 代码开源，安全机制透明
5. **持续改进**: 定期安全评估和更新
